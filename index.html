1 Аппаратная база и установка

🧱 1. Компоненты ПК (что и зачем)

  • Процессор (CPU): «Мозг». Характеристики: частота (ГГц), ядра, сокет.
  • Материнская плата: «Нервная система». Важна совместимость сокета с CPU и типа памяти (DDR).
  • Оперативная память (RAM): «Быстрая память». Энергозависима.
  • Накопители (SSD/HDD): «Долговременная память». SSD (быстрые, M.2/SATA) под систему, HDD (медленные, SATA) под файлы.
  • Видеокарта (GPU): Обработка графики. Дискретная (игры/работа) и встроенная (офис).
  • Блок питания (PSU): «Сердце». Питает всё.

🔌 2. Интерфейсы и BIOS/UEFI (как всё подключить и запустить)

  • SATA: Старый стандарт для HDD/SSD.
  • M.2: Современный разъём для скоростных NVMe SSD.
  • PCI-Express (PCIe): Слот для видеокарт и других плат расширения.
  • USB/HDMI/DP: Внешние подключения периферии и мониторов.
  • UEFI/BIOS: Микропрограмма для настройки оборудования до загрузки ОС. Вход при запуске ПК (клавиши Del, F2, F12 - зависит от производителя).

💿 3. Подготовка и установка Windows

Файловые системы:

  • NTFS: Основная для Windows. Поддерживает файлы любого размера, права доступа, шифрование, журналирование.
  • FAT32: Старый стандарт. Ограничение - файл не более 4 ГБ. Применяется для загрузочных флешек.
  • exFAT: Эволюция FAT32, нет ограничения в 4 ГБ. Хорошо подходит для внешних дисков и флешек (Windows, macOS, Linux).

Разметка диска:

  • MBR: Старый способ. Максимум 2 ТБ, 4 основных раздела. Только Legacy BIOS.
  • GPT: Современный. Нет ограничения в 2 ТБ, до 128 разделов. Обязателен для UEFI.

Создание загрузочной флешки:

  • Способ 1 (официальный): Media Creation Tool с сайта Microsoft.
  • Способ 2 (сторонний): Rufus. Выбрать ISO, схему раздела (GPT для UEFI, MBR для Legacy), файловую систему (FAT32 или NTFS).

Чистая установка (пошагово):

  1. Вставить флешку в USB-порт.
  2. Перезагрузить ПК, войти в UEFI/BIOS.
  3. В Boot Menu выбрать загрузку с USB-флешки.
  4. Дождаться запуска установщика Windows. Выбрать язык и раскладку.
  5. Выбрать «Выборочная: только установка Windows» - это чистая установка.
  6. Удалить все существующие разделы на целевом диске до «Незанятого пространства».
  7. Выбрать незанятое пространство и нажать «Далее».
  8. Дождаться завершения копирования файлов и перезагрузок.
  9. Пройти первоначальную настройку (регион, раскладка, учётная запись, конфиденциальность).
2 Администрирование Windows (клиент)

⚙️ 1. Системное ПО и драйверы

  • Драйверы: Программы-посредники между ОС и железом. Искать на оф. сайтах производителей (чипсет, видеокарта, сеть).
  • Точка восстановления: «Снимок» системы. Создавать перед установкой драйверов/софта.

👤 2. Учетные записи и безопасность

  • UAC (Контроль учётных записей): Запрос на подтверждение действий администратора. Не отключать!
  • Типы учеток: Администратор (полный доступ) и Стандартный пользователь (безопасный).
  • Создание учётной записи:
    • Параметры: Параметры → Учётные записи → Семья и другие пользователи → Добавить пользователя (локальная или Microsoft).
    • Управление компьютером: compmgmt.msc → Локальные пользователи и группы → Пользователи → ПКМ → Новый пользователь.
    • Командная строка: net user ИмяПользователя Пароль /add
  • Редактор локальной групповой политики: gpedit.msc. Настройка сложности паролей, блокировки экрана и т.д.

🧹 3. Реестр и автозагрузка (чистим систему)

  • Реестр (regedit): База данных настроек Windows. Иерархическая (разделы HKEY_*). Опасен для необдуманных правок.
  • Автозагрузка: Управляется в:
    • Диспетчере задач (вкладка «Автозагрузка»).
    • msconfig (вкладка «Автозагрузка»).
    • Планировщике заданий (taskschd.msc).
    • Папке автозагрузки: shell:startup (текущий пользователь) или C:\ProgramData\Microsoft\Windows\Start Menu\Programs\StartUp (все пользователи).

🔐 4. Права и общий доступ

Разрешения NTFS (локальный доступ):

  • Управляют доступом на уровне файловой системы - когда пользователь сидит прямо за этим компьютером.
  • Основные права: Чтение, Запись, Изменение, Полный доступ.
  • Настройка: ПКМ по папке → Свойства → вкладка «Безопасность» → Изменить.
  • Наследование: Права с родительской папки распространяются на вложенные.

Общие папки (SMB - доступ по сети):

  • Позволяют обращаться к папке с другого компьютера.
  • Настройка: ПКМ по папке → Свойства → вкладка «Доступ» → Общий доступ.
  • Сетевой путь: \\ИмяКомпьютера\ИмяОбщейПапки

Итоговое право (главное правило):

  • По сети работают и Share-права, и NTFS-права. Результирующее право - самое строгое из них.
  • Пример: Если на шаре «Полный доступ», а на NTFS «Чтение» → пользователь по сети сможет только читать.

📊 5. Диагностика и мониторинг

  • Диспетчер задач и Монитор ресурсов: Анализ загрузки CPU, памяти, диска. Поиск «виновников».
  • Службы (services.msc): Фоновые процессы ОС. Можно отключить ненужные для производительности.
  • Журналы событий (eventvwr): Главный инструмент для выяснения причин ошибок и BSOD (синих экранов).

🩹 6. Восстановление системы

  • Среда восстановления (WinRE): Загружается с установочной флешки.
  • Инструменты: Восстановление при загрузке (исправляет загрузчик), Откат к точке восстановления.
3 Сетевые технологии

🌐 1. Сетевое оборудование и физическая среда

Сетевое оборудование:

  • Концентратор (Hub): «Глупый» распределитель - дублирует сигнал на все порты. Создаёт коллизии. Почти не используется.
  • Коммутатор (Switch): «Умный» распределитель. Передаёт кадры только на нужный порт по MAC-адресу. Работает на L2. Бывают управляемые (с VLAN) и неуправляемые.
  • Маршрутизатор (Router): Связывает разные IP-сети. Принимает решение по таблице маршрутизации. Работает на L3.

Физическая среда (кабели):

  • Витая пара (Ethernet): Разъём RJ-45. Категории: Cat5e (1 Гбит/с), Cat6 (10 Гбит/с).
    • Прямой кабель: оба конца T568B. ПК ↔ коммутатор/роутер.
    • Перекрёстный кабель: один конец T568A, другой T568B. Сейчас почти не нужен (Auto-MDI/MDI-X).
  • Оптоволокно (Fiber): Сигнал светом. Огромные расстояния и скорости, защита от помех.

🛠️ 2. Базовая диагностика (первые шаги при «не работает интернет»)

  • ipconfig /all - все настройки IP.
  • ping 8.8.8.8 - проверка связи. Пинг на IP есть, на сайт нет → проблема с DNS.
  • tracert ya.ru - на каком этапе теряется пакет.
  • pathping - более детальная версия tracert.
  • nslookup ya.ru - проверка работы DNS.
  • netstat -ano - кто и на каких портах слушает систему.

📚 3. Теоретическая база (OSI и TCP/IP)

  • Модель OSI (7 уровней): Нужна для быстрой локализации проблемы. «Где обрыв: на 1-м (кабель) или на 7-м (неправильный URL)?»
  • Инкапсуляция: Данные «упаковываются» спускаясь по уровням (Данные → Сегмент TCP → Пакет IP → Кадр Ethernet).

📍 4. Адресация (IP и подсети)

  • MAC-адрес: Физический адрес сетевой карты (L2). Выглядит как 00-1A-2B-3C-4D-5E.
  • IP-адрес (IPv4): Логический адрес для маршрутизации (L3). Например, 192.168.1.10.
  • Маска подсети: Определяет границу сети.
    • /24 = 255.255.255.0 → сеть 192.168.1.0, хосты 192.168.1.1 – 192.168.1.254. Всего 254 хоста.
    • /25 = 255.255.255.128 → сеть 192.168.1.0, хосты 192.168.1.1 – 192.168.1.126. Всего 126 хостов.
  • Основной шлюз (Default Gateway): IP-адрес маршрутизатора для выхода в другие сети (например, 192.168.1.1).
  • Пример: Компьютер с IP 192.168.1.10/24 находится в сети 192.168.1.0/24. Чтобы отправить пакет на 8.8.8.8, он обращается к шлюзу 192.168.1.1.

🔑 5. Ключевые службы (критично для работы сети)

  • DHCP (Автоматическая выдача IP): Процесс DORA (Discover → Offer → Request → Acknowledge).
  • DNS (Система доменных имён): Преобразует ya.ru в IP-адрес. Иерархическая система.
  • Типы DNS-записей: A Имя → IPv4   AAAA Имя → IPv6   CNAME Псевдоним   MX Почтовый сервер.

🚦 6. Маршрутизация и NAT

  • Маршрутизатор: Связь разных IP-сетей. Таблица маршрутизации - route print.
  • Router-on-a-Stick: Один физический интерфейс роутера, разбитый на виртуальные (сабинтерфейсы) для маршрутизации между VLAN.
  • NAT/PAT: Технология, позволяющая многим устройствам с «серыми» IP выходить в интернет с одним «белым» IP.

🏷️ 7. VLAN и коммутация

  • VLAN (Виртуальные локальные сети): Разделение одного физического коммутатора на несколько логических, изолированных друг от друга.
  • Тегированный порт (Trunk): Передаёт трафик нескольких VLAN.
  • Нетегированный порт (Access): Порт в одном VLAN для конечного ПК.

🛡️ 8. Безопасность сети

  • ACL (Access Control Lists): Фильтрация трафика на маршрутизаторах/брандмауэрах.
  • Брандмауэр (Firewall): Правила на основе портов, протоколов и IP.
  • Wi-Fi безопасность: Только WPA2/3 с надёжным паролем. Гостевая сеть - отдельный SSID с изоляцией клиентов.

🚪 9. Удаленный доступ

  • RDP (3389/TCP): Штатное средство Windows для удалённого управления.
  • VPN: Защищённый «туннель» через небезопасный интернет. Позволяет удалённо подключиться к сети офиса (пример - GRE).
4 Windows Server (инфраструктура предприятия)

📦 1. Установка ролей и компонентов

  • Роли (Roles): Крупные функциональные блоки (AD DS, DNS Server, DHCP Server, Web Server).
  • Компоненты (Features): Вспомогательные функции (Telnet Client, .NET Framework, Windows Server Backup).
  • Установка:
    • Графический способ: Server Manager → Управление → Добавить роли и компоненты.
    • PowerShell: Install-WindowsFeature -Name ИмяРоли (например, AD-Domain-Services).

🏛️ 2. Active Directory (AD) и оснастка «AD - пользователи и компьютеры»

Active Directory - фундамент:

  • Домен: Группа ПК и пользователей с единой БД учётных записей (на контроллере домена).
  • Контроллер домена (DC): Сервер, где хранится AD.
  • OU (Organizational Unit): Контейнер для группировки объектов (пользователи, компьютеры).

Оснастка «AD - пользователи и компьютеры» (dsa.msc):

  • Создание пользователя: ПКМ по OU → Создать → Пользователь. Задать имя, логин, начальный пароль.
  • Создание группы: ПКМ по OU → Создать → Группа («Глобальная группа безопасности»). Затем добавить пользователей в группу (вкладка «Член групп» у пользователя или «Члены группы» у группы).

🎛️ 3. Групповые политики (GPO) - рычаг управления

  • Централизованная настройка сотен ПК: пароли, рабочий стол, деплой софта, запрет USB-флешек.
  • Инструмент: gpmc.msc (Group Policy Management Console).
  • Команды на клиенте: gpupdate /force (применить сейчас), rsop.msc (посмотреть применённое).

🌍 4. Сетевые службы в домене (DNS и DHCP на Windows Server)

DNS-сервер:

  • Устанавливается как роль «DNS-сервер». В домене часто ставится сразу с DC и настраивается автоматически.
  • Оснастка: dnsmgmt.msc.
  • Зоны: Зона прямого просмотра (corp.local), обратная зона.
  • Создание записи A: ПКМ по зоне → Создать узел (A) → ввести имя (например, intranet) и IP-адрес.
  • Проверка: nslookup intranet.corp.local должен вернуть IP.

DHCP-сервер:

  • Устанавливается как роль «DHCP-сервер». Обязательна авторизация в AD (ПКМ по серверу → Авторизовать).
  • Оснастка: dhcpmgmt.msc.
  • Создание области (Scope): ПКМ по IPv4 → Создать область. Задать диапазон адресов, маску, шлюз, DNS-серверы.
  • Резервация: ПКМ по «Резервирования» → Создать резервирование. Указать IP и MAC-адрес. Незаменимо для серверов и принтеров.
